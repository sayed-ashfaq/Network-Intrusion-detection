âš™ï¸ How It Works

1. Feature Extraction

The system extracts 30 features from each URL by analyzing:

  • URL structure and composition
  • Domain registration details via WHOIS lookups
  • SSL certificate validity and state
  • DNS records and reputation
  • HTML content and page structure

2. Data Preprocessing

Features are standardized using StandardScaler to normalize values between -1, 0, and 1, ensuring all features contribute equally to model training.

3. Dual Model Architecture

The system trains two complementary models:

XGBoost (Extreme Gradient Boosting): An ensemble method that builds a forest of decision trees sequentially, with each tree correcting errors from previous ones. Optimized for binary classification with regularization to prevent overfitting.

Configuration: 400 estimators, max_depth=7, learning_rate=0.04, strong regularization (alpha=0.3, lambda=1.0)
ANN/MLP (Artificial Neural Network - Multi-Layer Perceptron): A feedforward neural network with multiple hidden layers that learns non-linear relationships in the data through backpropagation. Uses ReLU activation for hidden layers and Adam optimizer.

Architecture: Input â†’ 128 neurons â†’ 64 neurons â†’ 32 neurons â†’ Output(Binary)
Features: Early stopping, validation split 0.1, batch size 64, max iterations 400

4. Model Evaluation & Selection

Both models are evaluated on the test set using:

  • Accuracy: Overall correctness of predictions
  • Precision: Of detected phishing sites, how many are true positives
  • Recall: Of actual phishing sites, how many are detected
  • F1-Score: Harmonic mean balancing precision and recall

ğŸ“š Essential Definitions

Machine Learning Concepts

Supervised Learning: Training algorithm on labeled data (input-output pairs) to learn patterns and make predictions on new, unseen data. Used here to classify URLs as phishing or legitimate based on historical examples.
Classification: A machine learning task where the model predicts discrete categories (in this case: Phishing or Legitimate) rather than continuous values.
Feature Engineering: The process of creating meaningful features from raw data that help the model learn patterns. Here, we extract URL characteristics, domain information, and page structure features.
Cross-Validation: A technique to evaluate model performance by dividing data into k-folds, training on k-1 folds, and testing on the remaining fold. Provides robust performance estimates.

XGBoost (Extreme Gradient Boosting)

Definition: An optimized gradient boosting framework that builds an ensemble of decision trees. Each new tree is trained to correct residuals (errors) from previous trees, creating a powerful combined model.
Key Advantages:
  • Fast training and prediction
  • Handles non-linear relationships well
  • Built-in regularization prevents overfitting
  • Robust to outliers in data
  • Excellent performance on tabular data

ANN (Artificial Neural Network) / MLP (Multi-Layer Perceptron)

Definition: A feedforward neural network with multiple layers of interconnected neurons. Each neuron performs weighted sum of inputs + bias, followed by activation function. The network learns by adjusting weights through backpropagation.
How It Works:
  • Input Layer: Receives 30 features
  • Hidden Layers: Learn hierarchical representations (128 â†’ 64 â†’ 32 neurons)
  • Output Layer: Binary classification (Phishing vs Legitimate)
  • Activation: ReLU introduces non-linearity in hidden layers
  • Backpropagation: Adjusts weights to minimize prediction error

Evaluation Metrics

Accuracy: (TP + TN) / Total. Percentage of correct predictions. Good overall metric but can be misleading with imbalanced data.
Precision: TP / (TP + FP). Of all URLs flagged as phishing, how many were actually phishing? Critical to avoid false positives.
Recall: TP / (TP + FN). Of all actual phishing URLs, how many were detected? Critical to avoid missing real threats (false negatives).
F1-Score: 2 Ã— (Precision Ã— Recall) / (Precision + Recall). Harmonic mean of precision and recall, best for imbalanced datasets.
